# تقرير تدريب نموذج توقع البنود

## البيانات

- عدد المشاريع المعتمدة حسب توجيه المستخدم: **340**.
- عدد مجموعات المشاريع: **17**، بواقع 20 مشروعًا لكل مجموعة.
- عدد أكواد البنود الفريدة: **203**.
- الاستخدام: توقع حضور البنود وتصنيفها فقط، وليس تدريب الكميات أو الأسعار.

## التقسيم والتدريب

- Training: 230 مشروعًا.
- Validation: 59 مشروعًا.
- Test: 34 مشروعًا.
- Final Holdout: 17 مشروعًا.
- التقسيم حتمي باستخدام `content_hash`، مع منع تكرار السجل بين المجموعات.
- تم تنفيذ جولتين من Hard Negative Mining، وأضيفت 8 حالات صعبة.
- بعد التقييم، أعيد تدريب النموذج النهائي على جميع المشاريع الـ340 للاستخدام في التطبيق.

## الربط بالواجهة

المسار الفعلي أصبح:

الوصف → فهم المشروع → نموذج حضور البنود → تصنيف البنود → قواعد النفي والتعارض → مراجعة البنود → `approvedBoq`.

لا يحفظ التطبيق تلقائيًا إلا البنود المصنفة `core` والمتوافقة كذلك مع محرك الكميات. تعرض صفحة المشروع شارة إصدار نموذج البنود المستخدم.
